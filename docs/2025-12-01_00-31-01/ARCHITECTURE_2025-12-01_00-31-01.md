## System Overview
School Buddy consists of three coordinated layers:

---

## 1. FRONTEND (React)
**Location:** `client/`

### Key responsibilities
- Fetch school + catchment data from backend
- Render map with:
  - Leaflet
  - Turf.js (geometry operations)
- Display school markers, circles, and GeoJSON boundaries
- Allow selection of multiple schools

### Important components
- `App.jsx`
- `MapView.jsx`
- `SchoolSelector.jsx`

---

## 2. BACKEND (Node.js + Express)
**Location:** `server/`  
**Entry point:** `server/src/index.js`

### Responsibilities
- Serve REST API  
- Retrieve schools from PostgreSQL  
- Load GIS GeoJSON files from disk  
- Serve Tiffin catchment data  
- Provide scalable structure for adding more catchment routes

### Routes
- `GET /api/schools`
- `GET /api/schools/catchment/tiffin`

---

## 3. DATABASE (PostgreSQL)
### Core table: `schools`
Stores:
- Coordinates  
- Radius-based catchment (some schools)
- GeoJSON polygons (others)
- Names, IDs, metadata

---

## 4. GIS PIPELINE
**Location:** `gis/`  
Processes:
- Raw postcode list  
- GeoJSON boundary shaping  
- Output polygons:
  - `tiffin_boundary.geojson`
  - `tiffin_individual.geojson`

These files are read directly by Express.

---

## High-Level Data Flow
1. Frontend loads → fetches 
2. User selects a school → frontend:
   - draws marker
   - draws circle (if radius)
   - fetches polygon catchments (if polygon-based)
3. Backend reads DB or GeoJSON files
4. Leaflet renders everything

---

## Scalability Notes
- Adding new schools requires only:
  - A DB insert
  - Optionally adding a new /api/schools/catchment/<school> route
- Polygons are stored as static GeoJSON for performance
- Radius-based catchments use Leaflet circle rendering
